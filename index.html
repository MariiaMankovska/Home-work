Основні команди GIT 🤯

  • git config --global user.name - вказати ім'я;
  • git config --global user.email - реєстрація пошти;
  • git init - створення нового репозиторію;
  • git status - показує стан файлів в робочому каталозі: які файли змінені, але не добавлені в індекс; які очікують коміту в індексі;
  • git add - добавляє вміст робочого каталогу в індекс для наступного коміту;
  • git commit - записує зміни до сховища;
  • git log - показує історію комітів виконаних раніше;
  • git checkout - команда для переходу між гілками;
  • git branch - для роботи з гілками;
  • git push - для вигрузки з локального у віддалений репозиторій. Відправляє на сервер;
  • git pull - для загрузки із віддаленого репозиторія на локальний;
  • git merge - для зєднування однієї або кількох гілок в основну;
  • git clone - для клонування (копіювання) репозиторію;

    • advanced 🧐

  • git revert - для безпечної відміни змін без втрати історії комітів;
  • git reset - для відміни локальних змін в репозиторії;
  • git rebase - ще один спосіб перенести зміни з однієї гілки на іншу;
  • git cherry-pick - для перенесення окремих комітів з одного місця репозиторію в інше, зазвичай між гілками;
  • git stash - дозволяє тимчасово "здати в архів" зміни, зроблені в робочій копії, щоб ми могли застосувати їх пізніше.
